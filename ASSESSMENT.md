## 1. Repository Purpose

**SzyCo.Garage** is a web-based garage/vehicle management system that enables users to:
- Track personal vehicle collections (make, model, year, color)
- Manage user accounts with role-based access control
- View comprehensive audit logs of all data changes
- Administer users, roles, and permissions (admin functions)

The system demonstrates Coalesce framework capabilities including:
- Automatic API generation from C# models
- TypeScript client code generation
- Vue 3 UI scaffolding with Vuetify 3
- Audit logging with minimal custom code

### 2.3 Backend Architecture (.NET)

**SzyCo.Garage.Data Project:**
- **Models:** Car, User (extends IdentityUser), Role, UserRole, AuditLog, Widget
- **DbContext:** AppDbContext (extends IdentityDbContext + audit logging)
- **Base Classes:**
  - `TrackingBase`: Provides CreatedBy/CreatedOn, ModifiedBy/ModifiedOn audit fields
  - `AppDataSource<T>`: Generic wrapper for Coalesce data sources
  - `AppBehaviors<T>`: Generic wrapper for save/delete behaviors
- **Custom Data Sources:**
  - `Car.MyGarage`: Filters cars to current user only
  - `User.DefaultSource`: User-specific query filtering
  - `UserRole.DefaultSource`: Role assignment filtering
- **Services:**
  - `CarService`: Vehicle management operations
  - `UserManagementService`: Email/password operations
  - `SecurityService`: Permission checks and user info

**SzyCo.Garage.Web Project:**
- **Entry Point:** Program.cs (minimal hosting model)
- **Configuration:** 
  - SQL Server with retry logic
  - ASP.NET Identity with custom claims
  - Coalesce API generation
  - Static file caching for Vite bundles
- **Controllers:**
  - `HomeController`: Serves SPA index.html (requires authentication)
  - Generated API controllers (in Api/Generated/): CarController, UserController, RoleController, etc.
- **Authentication:**
  - Cookie-based with 5-minute security stamp validation
  - Password policy: 15-char minimum (NIST 800-63-4 compliant)
  - Custom claim types: Role, Email, UserId, UserName

### 2.4 Frontend Architecture (Vue 3)

**Structure:**
```
src/
├── views/              # Page components
│   ├── Car.vue        # Main garage listing
│   ├── CarMain.vue    # Car detail view
│   ├── Admin.vue      # Admin dashboard
│   └── UserProfile.vue
├── components/         # Reusable components
│   ├── CarForm.vue
│   ├── CarTable.vue
│   └── CarHero.vue
├── composables/        # Vue 3 composition utilities
├── *.g.ts             # GENERATED by Coalesce (DO NOT EDIT)
├── router.ts          # Vue Router configuration
└── user-service.ts    # Global auth state
```

**Generated Files Pattern:**
- All files ending in `.g.ts` or `.g.cs` are generated by Coalesce
- Generated files include: models, viewmodels, API clients, metadata
- **Never edit generated files manually**

**Key Frontend Features:**
- Vuetify 3 Material Design components
- Vue Router with permission-based route guards
- Auto-refresh authentication (every 5 minutes)
- TypeScript type safety with generated models
- Vite HMR for fast development

### 2.5 Data Flow

1. **Model Definition** (C#) → Coalesce Attributes
2. **Code Generation** → `dotnet coalesce` generates TypeScript + API controllers
3. **API Layer** → RESTful endpoints at `/api/{Entity}`
4. **Vue Client** → Calls APIs via generated clients
5. **State Management** → Vue 3 reactivity + viewmodels
6. **UI Rendering** → Vuetify components

### 2.6 Authentication & Authorization Flow

1. User accesses protected route → redirected to `/SignIn`
2. Login via ASP.NET Identity → cookie issued
3. Frontend calls `SecurityService.whoAmI()` → fetches user + permissions
4. Permission checks in:
   - Backend: `[Edit(Permission.UserAdmin)]` attributes
   - Frontend: `can(Permission.*)` helper in user-service.ts
5. Auto-refresh: Every 5 minutes + on window focus
6. 401 response → triggers page reload to re-authenticate

### 2.7 Generated vs Custom Code

| Generated (DO NOT EDIT) | Custom (Editable) |
|---|---|
| Models/Generated/*.cs | Models/*.cs |
| Api/Generated/*.cs | Custom controllers |
| src/*.g.ts | src/views/*.vue |
| | src/components/*.vue |
| | src/router.ts |
| | src/composables/*.ts |
| | Data sources, behaviors |

**Important:** Coalesce regenerates `.g.` files on every build. Custom logic must go in:
- Partial classes
- Data source overrides
- Behavior classes
- Custom services
- Vue components (not generated)
